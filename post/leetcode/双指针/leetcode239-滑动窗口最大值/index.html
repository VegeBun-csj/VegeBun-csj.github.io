给定一个大小为 n ≤ $10^{6}$的数组。

有一个大小为 k 的滑动窗口，它从数组的最左边移动到最右边。

您只能在窗口中看到 k 个数字。

每次滑动窗口向右移动一个位置。

以下是一个例子：

该数组为 [1 3 -1 -3 5 3 6 7] ，k为3。输入格式

输入包含两行。

第一行包含两个整数 n 和 k ，分别代表数组长度和滑动窗口的长度。

第二行有 n 个整数，代表数组的具体数值。

同行数据之间用空格隔开。

输出格式

输出包含两个。

第一行输出，从左至右，每个位置滑动窗口中的最小值。

第二行输出，从左至右，每个位置滑动窗口中的最大值。

输入样例：

1
2

8 3
1 3 -1 -3 5 3 6 7

输出样例：

1
2

-1 -3 -3 -3 3 3
3 3 5 5 6 7

#include<iostream>
using namespace std;
const int maxn = 1000010;
//a数组存值，q数组存下标
int a[maxn],q[maxn];
int n,k;
int main(){   
    cin >> n >> k;
    for(int i = 0;i < n;i ++) cin >> a[i];
    
    int h = 0,t = -1;
    for(int i = 0;i < n;i ++){
        if(h <= t && i - k + 1 > q[h]) h ++;
        while(h <= t && a[q[t]] >= a[i]) t --;
        q[++ t] = i;
        if(i >= k - 1) cout << a[q[h]] << " ";
    }
    cout << endl;
    
    h = 0, t = -1;
    for(int i = 0;i < n;i ++){
        if(h <= t && i - k + 1 > q[h]) h ++;
        while(h <= t && a[q[t]] <= a[i]) t --;
        q[++ t] = i;
        if(i >= k - 1) cout << a[q[h]] << " ";
    }
    return 0;
}

  • 朴素的做法：

    对窗口中的K个元素进行遍历，找到最值元素；然后每移动一次窗口，遍历一次窗口内的元素，得到最值。这样总的时间复杂度就是O(KN)，就不是线性的复杂度了，时间复杂度比较高。

  • 使用一个单调队列，同时又是一个双端队列，实现求滑动窗口中的最值。大体思路就是用一个队列来维护一个数组元素下标，并且这个队列中的元素下标所对应的值是单调的，要找窗口最值元素的时候，只需要输出队头即可.

    • 如果要寻找窗口的最大值，那么将当前队尾元素下标对应的值小于当前遍历到的值，那么当前队列中的下标就要让位，让当前遍历到的这个元素下标进队列，因为对于寻找最大值，队尾元素下标对应的值是不可能成为最大值的，因为后面这个更大的进来了。
    • 如果要寻找窗口的最大值，那么将当前队尾元素下标对应的值大于当前遍历到的值，那么当前队列中的下标就要让位，让当前遍历到的这个元素下标进队列，因为对于寻找最小值，队尾元素下标对应的值是不可能成为最小值的，因为后面这个更小的进来了。

题目描述

给定一个数组 nums，有一个大小为 k 的滑动窗口从数组的最左侧移动到数组的最右侧。你只可以看到在滑动窗口内的 k 个数字。滑动窗口每次只向右移动一位。

返回滑动窗口中的最大值。

进阶：

你能在线性时间复杂度内解决此题吗？

class Solution {
public:
    vector<int> maxSlidingWindow(vector<int>& nums, int k) {
        //结果数组
        vector<int> res;
        //队列存下标
        deque<int> q;
        for(int i = 0;i < nums.size();i ++){
            if(q.size() && i - k + 1 > q.front()) q.pop_front();
            while(q.size() && nums[q.back()] <= nums[i]) q.pop_back();
            q.push_back(i);
            if(i >= k - 1) res.push_back(nums[q.front()]);
        }
        return res;
    }
};

复杂度分析

时间复杂度为O(N)，空间复杂度为O(N)；

注意每次出队的时候找到最值元素的时间复杂度为O(1)，只不过要找到N个最值元素，所以才是O(N)
