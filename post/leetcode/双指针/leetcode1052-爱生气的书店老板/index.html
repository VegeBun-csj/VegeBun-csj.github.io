题目描述

今天，书店老板有一家店打算试营业 customers.length 分钟。每分钟都有一些顾客（customers[i]）会进入书店，所有这些顾客都会在那一分钟结束后离开。

在某些时候，书店老板会生气。 如果书店老板在第 i 分钟生气，那么 grumpy[i] = 1，否则 grumpy[i] = 0。 当书店老板生气时，那一分钟的顾客就会不满意，不生气则他们是满意的。

书店老板知道一个秘密技巧，能抑制自己的情绪，可以让自己连续 X 分钟不生气，但却只能使用一次。

请你返回这一天营业下来，最多有多少客户能够感到满意的数量。

输入：customers = [1,0,1,2,1,1,7,5], grumpy = [0,1,0,1,0,1,0,1], X = 3

输出：16

解释： 书店老板在最后 3 分钟保持冷静。 感到满意的最大客户数量 = 1 + 1 + 1 + 1 + 7 + 5 = 16.

题目链接

https://leetcode-cn.com/problems/grumpy-bookstore-owner

思路

滑动窗口

grump数组代表的是老板生不生气，如果老板生气，顾客就不高兴；反之顾客高兴。所以客户数量中肯定要包含老板不生气的时候顾客的数量，然后老板是由保持冷静的技能的，要求出能达到最大客户数量的效果，必须要将这个保持冷静的技能用在刀刃上，那这个刀刃是什么呢？仔细思考一下这个技能肯定要在人数最多的时间发动才行，这样才能保证最多的人满意，客户不流失嘛。所以这个才是决定人数最多的关键，只有将尽可能多的不满意的客户转变为满意的客户，这样才能达到最多满意的客户。

所以可以将客户不满意的人数放在数组中，然后使用滑动窗口，窗口大小为 X，在窗口向右滑动的过程中，依次计算每个窗口中所有值的和，同时记录下左右窗口的最大值，这个最大值是老板可以发动技能让最多客户满意的数量，最终的最大客户满意数量=冷静技能发动后满意的最大客户数量 + 老板开心的时候所有客户的数量

class Solution {
public:
    int maxSatisfied(vector<int>& customers, vector<int>& grumpy, int X) {
        int res = 0;
        for(int i = 0;i < customers.size();i ++){
            if(! grumpy[i]) res += customers[i];
            else grumpy[i] = customers[i];
        }

        //tempSum用来记录每个窗口中的值的和，Max记录的是所有窗口中最大的值
        int tempSum = 0,Max = 0;
        for(int i = 0;i < X;i ++) tempSum += grumpy[i];

        Max = tempSum;

        //窗口后移，计算出每个窗口的总共值，然后取最大的即可，最大的就是老板发动技能的时机
        for(int i = X;i < grumpy.size();i ++){
            tempSum -= grumpy[i - X];
            tempSum += grumpy[i];
            if(tempSum > Max) Max = tempSum;
        }

        return res + Max;
    }
};

复杂度分析

时间复杂度为O(N)，空间复杂度为O(1)
