题目描述

反转从位置 m 到 n 的链表。请使用一趟扫描完成反转。

说明:

1 ≤ m ≤ n ≤ 链表长度。

示例:

输入: 1->2->3->4->5->NULL, m = 2, n = 4

输出: 1->4->3->2->5->NULL

题目链接

https://leetcode-cn.com/problems/reverse-linked-list-ii

思路

一般而言，对于链表的题目，都需要创建一个dummpy节点，用来指向首节点，从而方面对链表首节点的操作；

本题和反转链表 还不一样，要求更多一点，即反转链表中的一部分；首先知道，要操作链表中的节点必须要知道这个节点的上一个节点才行，所以如果是对于反转第m到第n个节点，那么必须要知道第m-1个节点才行，所以:

  • 第一步就是寻找第m-1个节点，设置一个pre指针指向dummpy节点，要找到第 m - 1个节点，pre需要移动 m-1 步（这个可以在脑子里演示一下：比如 m = 1，即从第一个节点开始反转，那么就需要找到第一个节点的前一个节点，现在pre 指向dummpy，那么pre需要移动 1 - 1 = 0 步，即不需要移动， 因为dummpy本来就在第1个节点前面；又比如 m = 3，即从第3个节点开始反转，那么需要找到第2个节点话，pre需要移动 3 - 1 = 2 步，即从指向 dummpy 转而指向第1个，再从第1个指向第2个，所以就两步；这里就可以看出dummpy节点的好处了就是将对首节点的操作与后面的操作一致）
  • 再看我们反转的过程，将1->2->3->4->5->NULL变成 1->3->2->4->5->NULL，然后再变成1->4->3->2->5->NULL，首先看2 -> 3转变为3->2的过程，此时pre是指向节点1的，我们设置一个cur指针指向2，然后再设置一个指针temp指向3，因为待会要将2和3进行断开，为了防止断链，所以用temp记住节点3，然后操作就是将2->4，即cur -> next = temp -> next，然后就是将3->2 ，即temp -. next = pre -> next，然后将1->3，即pre -> next = temp；后面的变换同样的操作
  • 这个交换需要多少次呢？可以看到 m = 2，n = 4 ，实际只需要2次，即 n - m次即可

class Solution {
public:
    ListNode* reverseBetween(ListNode* head, int m, int n) {
        ListNode* dummpy = new ListNode(0);
        dummpy -> next = head;
        ListNode* pre = dummpy;
        //pre指向第m个节点的前一个节点
        for(int i = 0;i < m - 1;i ++){
            pre = pre -> next;
        }

        ListNode* cur = pre -> next;

        for(int i = m;i < n;i ++){
            ListNode* temp = cur -> next;
            cur -> next = temp -> next;
            temp -> next = pre -> next;	//这里不能写成cur
            pre -> next = temp;
        }
        return dummpy -> next;
    }
};

注意：上面节点进行交换的时候 temp -> next = pre -> next不能写成temp -> next = cur，这是一个错误，可以画图看出来

可以看到，在整个节点的交换中，pre是始终指向第n个节点的前一个节点1的，只有cur在不断向后变动，temp不断更新为cur的下一个节点

复杂度分析

时间复杂度为O(N)，空间复杂度为O(1)
