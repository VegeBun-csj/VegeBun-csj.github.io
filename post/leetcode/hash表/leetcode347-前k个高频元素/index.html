题目描述

给定一个非空的整数数组，返回其中出现频率前 k 高的元素。

示例 1:

输入: nums = [1,1,1,2,2,3], k = 2 输出: [1,2] 示例 2:

输入: nums = [1], k = 1 输出: [1]

题目链接

https://leetcode-cn.com/problems/top-k-frequent-elements

思路一(暴力解法–hash表+排序法)

hash表 + 普通排序

  • 遍历数组，然后将其中的元素存入hash表，元素值作为hash表的键，元素出现的次数作为hash表的值.这样hash表就可以进行去重
  • 将hash表中的元素进行排序（这里是用了一个vector<pair<int,int»辅助来对hash中的元素排序的），然后从这个vector中截取前K个元素，返回

class Solution {
public:
    static bool cmp1(pair<int,int> a,pair<int,int> b){
        return a.second > b.second;
    }

    vector<int> topKFrequent(vector<int>& nums, int k) {
        vector<pair<int,int>> res;
        vector<int> ans;
        unordered_map<int,int> nums_map;
        for(auto n : nums) nums_map[n] ++;
        
        //遍历hashmap
        auto iter = nums_map.begin();
        while (iter!= nums_map.end()){  
            res.push_back(make_pair(iter->first,iter->second));
            ++iter;  
        }  

        sort(res.begin(),res.end(),cmp1);

        for (auto it = res.begin(); it != res.end(); it++){
            ans.push_back(it -> first);
        }
        vector<int>::const_iterator first = ans.begin();
        vector<int>::const_iterator end = ans.begin() + k;
        vector<int> r(first,end);

        return r;
    }
};

复杂度分析

时间复杂度为O(NlogN)，空间复杂度为O(N)，这个解法的时间复杂度不满足题目的要求，因为要求时间复杂度不能高于NlogN，主要是对于hash表中所有的元素进行排序，然后截取了前K个，那么能不能就只针对K个元素进行排序呢？这里可以用堆来实现

思路二(hash表+堆)

hash表 + 堆

  • 遍历数组，然后将其中的元素存入hash表，元素值作为hash表的键，元素出现的次数作为hash表的值.这样hash表就可以进行去重
  • 建立小根堆（堆顶为堆的最小元素），当堆的元素个数小于K的时候，将hash表中的键值存放进堆（注意当如堆的时候，应该将元素出现的次数作为堆的中元素的键，将元素值作为堆中元素的值,为什么这么放？因为我们是要找出现频次最高的k个数，所以堆中应该是以频次进行排序的，而c++中的优先队列建立的堆是以第一个元素来排序的，所以将频次作为第一个元素），当到第k + 1个元素的时候，比较其频次与堆顶元素频次的大小，如果比堆顶元素小，那么遍历hash表的下一个；如果比堆顶元素的频次大，那么将堆顶元素pop()，然后将该元素及其频次放入堆中
  • 遍历完hash表，然后用一个数组来记录堆中的元素，输出即可

class Solution {
public:
    vector<int> topKFrequent(vector<int>& nums, int k) {
        unordered_map<int,int> nums_map;
        //塞进hash表
        for(auto n : nums) nums_map[n] ++;
        //hash表中元素为[数值，频次]]

        //堆中的元素为 [频次，数值]
        //堆中是按照频次，即以first进行排序的，最后输出的时候只需要输出数值即可
        priority_queue<pair<int,int>,vector<pair<int,int>>,greater<pair<int,int>>> q;

        for(auto it : nums_map){
            if(q.size() != k) q.push(make_pair(it.second,it.first));
            else{
                if(it.second > q.top().first){
                    q.pop();
                    q.push(make_pair(it.second,it.first));
                }
            }
        }

        vector<int> res;
        while(! q.empty()){
            res.push_back(q.top().second);
            q.pop();
        }
        //因为是小根堆，所以频次小的先输出
 		reverse(res.begin(),res.end());
        return res;
    }
};

复杂度分析

时间复杂度为O(NlogK)，空间复杂度为O(N)

这里一个优化就是堆排序，并不需要对所有元素排序，只需要对堆中的K个元素排序即可
